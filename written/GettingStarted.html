Getting Started with the Daily.co API

Ship 1-click video chat with our API that allows you to add video chat to any app or site (web and mobile), in minutes. Our calls are WebRTC, and work in the browser.

Our goals for the Daily.co API are:

  1. To get you up and running with video calls in just a few minutes.
  2. To enable a wide variety of use cases and workflows.
  3. To give you a complete, ready to go, user interface.

Quickly Adding a Call to a Web page

The snippet below displays a Daily.co call in the lower right area of the page, but this is only a starting point- you can customize everything about the call.

  • HTML
<script crossorigin src="https://unpkg.com/@daily-co/daily-js"></script>
<script>
callFrame = window.DailyIframe.createFrame();
callFrame.join({ url: 'https://your-team.daily.co/hello' })
</script>

For example, the code below returns a Daily.co call as a full-page video call.

  • HTML
<script crossorigin src="https://unpkg.com/@daily-co/daily-js"></script>
<script>
callFrame = window.DailyIframe.createFrame({
  showLeaveButton: true,
  iframeStyle: {
    position: 'fixed',
    width: '100%',
    height: '100%'
  }
});
callFrame.join({ url: 'https://your-team.daily.co/hello' })
</script>


Creating Rooms (A walk-through of creating rooms using our REST API- delete)

There are many ways to creating rooma using the REST API.

  1. Via https://dashboard.daily.co/
  2. Via a (Daily.co helper script)[https://github.com/daily-co/daily-demos/blob/master/static-demos/shared-assets/create-demo-room.js] to create a demo room under the api-demo.daily.co domain.
  3. Via the API docs
  1. Via your own script, for example:
  • JavaScript
var request = require("request");

var options = {
  method: 'POST',
  url: 'https://api.daily.co/v1/rooms/room_name',
  headers: {'content-type': 'application/json', authorization: 'Bearer '}
};

request(options, function (error, response, body) {
  if (error) throw new Error(error);

  console.log(body);
});

Under the hood our API creates an Iframe to embed a video call into your application. If you don't want to write any code at all, you can send Daily.co video call room links to your users directly.

IFrames, Mobile Devices, and Browser Support

Mobile Suport

  • iOS 12.1 and above
  • Android 5.0 and above (with current security and platform updates)

iOS Development

  • If you are making a call within a native application, you have to open a Safari tab from your application, because the native iOS WebView component does not (yet) allow access to the necessary camera, microphone, and network protocols.
  • If you are starting calls from a web application on iOS, it usually makes sense to open a new tab because mobile device screen sizes tend to be small enough that there's not much room to embed the call as an iframe.

We provide a couple of configuration parameters that help streamline the user flow into and out of a new Safari tab.

Android

On Android, you can embed calls inside a WebView in your application.

For more information on embedding Daily.co calls in native mobile apps, please see (this blog post)[https://www.daily.co/blog/mobile-video-chat-for-ios-and-android-app].

Browser Support

  • Chrome 74 and above
  • Safari 12.1 and above
  • Firefox 66 and above
  • Microsoft Edge 18 - only 1:1 call functionality, group calls are not supported
  • Electron 6 and above

Chrome is the only browser that can screen-share. (All browsers can view screen shares, but only a call participant using Chrome can share their screen.)

Scalability

Technical Considerations For Scalable Design and Implementation

Building an application for a few users is different than for tens as is different for hundreds.

From a technical standpoint, the Daily.co API supports up to 10,000 programmatic rooms and each room up to 200 participants. Daily.co video is built on top of a standard called WebRTC. WebRTC allows you to send streams of data (such as video and audio) between any two computers on the Internet. WebRTC works inside the browser for video capabilities, while amazing this also means that in a large room where everyone has turned on camera and microphone functionality, the browser will undoubtedly be overloaded.

Unfortunately, in this case, "a large room" is not in fact that large- we advise that only 6-8 cameras be on at any given time to give your application a standing chance. This number varies depending on variables like devices and networks but it is a good number to keep in mind nonetheless.

Fortunately Daily.co has several solutions (and more to come) for your scalable needs. The following are useful considerations:

  • Turn on broadcast mode via the owner_only_broadcast property and set the is_owner property where applicable. Daily.co's owner_only_broadcast property enables only meeting owners to send audio and video. Turning on this setting would disable the camera, microphone, and screen sharing functionality for others and will not display their respective buttons in the menu bar.
  • Make it so that when a user joins a room by default their video and audio are off- programmatically turning on their camera only in the event that they "raise their hand" and off when their turn has finished.
  • Turn off chat functionality but allow camera functionality.

HIPPA Compliance

Getting Started With HIPPA Compliance

Turning on compliance requires the following:

  1. Signing up at Daily.co and claiming your subdomain
  2. Generating your (always free) API key
  3. Contacting Us Via Form (Here)[https://dailyco.typeform.com/to/OZnPSE] where you can tell us more about what domain you want compliance for and so that we can touch base with you on BAA.
  4. Use our (Front-End library)[https://docs.daily.co/reference#using-the-dailyco-front-end-library] to embed calls in your HIPPA Compliant application

Feel free to contact us with any questions. We would love to help you get started!
